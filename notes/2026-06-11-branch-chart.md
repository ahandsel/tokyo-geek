## What was added


### 1. Theme customisations (commit `93e1b62`)

Rewrote `docs/.vitepress/theme/index.ts` to wire in three things the current `contents/.vitepress/theme/index.ts` still does NOT have:

* **ImageViewer activator** - boots `@davidingplus/vitepress-image-viewer` after hydration, with SVG-skip logic via a `no-viewer` class and a `MutationObserver` that re-marks SVGs as new nodes appear. See [`branch-chart/ImageViewerActivator.vue`](./branch-chart/ImageViewerActivator.vue).
* **Mermaid renderer** - initialises `vitepress-mermaid-renderer` inside the `Layout` setup so it reacts to the VitePress `isDark` switch (`forest` + `handDrawn` in light mode, `dark` + `default` in dark mode). The companion stylesheet [`branch-chart/vitepress-mermaid-renderer.css`](./branch-chart/vitepress-mermaid-renderer.css) is imported manually for Cloudflare Pages compatibility.
* **VideoViewer** - global Vue component registered via `enhanceApp`, with lazy loading via `IntersectionObserver` (200 px root margin). See [`branch-chart/VideoViewer.vue`](./branch-chart/VideoViewer.vue).

Also adds slot overrides to disable the image viewer on the home-page hero and feature images via a `no-viewer` class.

Full diff: [`branch-chart/theme-index-diff.patch`](./branch-chart/theme-index-diff.patch).

**Why apply:** AGENTS.md still lists `vitepress-image-viewer`, `VitePress Mermaid Renderer`, and the readabilities plugin as part of the stack, but the current `contents/.vitepress/theme/index.ts` is a near-empty default theme. These three pieces appear to be missing wiring.


### 2. Chart.js plugin wiring (commit `0656dda`)

Wraps the VitePress config in `withChartjs(...)` and adds the runtime style import:

```ts
// contents/.vitepress/config.mts
import { withChartjs } from 'vitepress-plugin-chartjs';
// ...
export default withChartjs(
  defineConfig(withSidebar(vitePressOptions, vitePressSidebarConfigs)),
);
```

```ts
// contents/.vitepress/theme/index.ts
import 'vitepress-plugin-chartjs/style.css';
```

Plus `package.json` deps: `chart.js@^4.5.1`, `vitepress-plugin-chartjs@^1.2.1`.

**Caution:** PRs #81 and #82 attempted exactly this and were closed without merging. Do not re-add Chart.js without a concrete page that needs it.
